# 全球热点地图产品方案（每 15 分钟自动更新）

## 1. 目标
构建一个系统：每 15 分钟自动从多个信息源抓取“全球热点新闻与局势动态”，经过清洗与评分后，在世界地图上可视化，并自动调整地图下方信息栏（重点事件、风险提示、影响范围、建议关注方向）。

---

## 2. 核心能力拆解

### 2.1 数据采集（Ingestion）
- **更新频率**：固定 15 分钟一次（可配置）。
- **数据来源建议**：
  - 权威媒体 RSS/API（国际新闻、地区新闻）。
  - 政府/国际组织公开通告源（外交、灾害、公共卫生）。
  - 金融/航运/安全类公开动态源（可选）。
- **采集方式**：
  - 优先官方 API；无 API 用 RSS；再考虑合法网页抓取。
- **关键元数据**：标题、正文摘要、发布时间、来源可信度、地理信息（国家/城市/经纬度）。

### 2.2 内容处理（NLP Pipeline）
- **去重聚类**：同一事件多源报道合并为“事件簇”。
- **主题分类**：战争冲突、灾害、公共卫生、政治外交、经济金融等。
- **地理解析**：
  - 用 NER 抽取地点。
  - 地名标准化（同名地歧义消解）。
  - 映射到经纬度。
- **态势评分（Hotness Score）**：
  - 可由“来源权重 × 时效性 × 传播度 × 事件严重度”计算。
  - 输出 0~100 分用于地图热度与排序。
- **情绪/风险标签**（可选）：
  - 紧张升级、缓和、预警、已确认等。

### 2.3 地图可视化
- **展示层级**：全球 -> 洲 -> 国家 -> 城市。
- **可视元素**：
  - 热点点位（点大小表示热度）。
  - 热区覆盖（区域多事件叠加）。
  - 事件时间轴回放。
- **交互行为**：
  - 点击点位查看事件卡片（来源、摘要、时间、影响评级）。
  - 筛选（时间范围、事件类型、风险等级、地区）。

### 2.4 信息栏自适应（你提到的关键点）
信息栏可以根据当前地图焦点和全局态势自动变化，建议分三层优先级：
1. **全局 Top 5 热点**（默认）。
2. **当前视野相关热点**（用户地图缩放/拖动后优先显示该区域事件）。
3. **用户关注主题**（如“中东局势”“海运风险”“能源供应”）。

动态规则示例：
- 若某区域 1 小时内热度增幅 > 阈值，信息栏顶端插入“突发升级”。
- 若多个来源一致提到同一事件，提升其“可信优先级”。
- 若事件已持续多周期但无增量，自动降权，避免信息栏被“旧闻”占据。

---

## 3. 推荐技术架构（MVP 可落地）

### 3.1 后端服务
- `collector-service`：定时采集（Cron/队列调度）。
- `nlp-service`：抽取、分类、聚类、评分。
- `geo-service`：地名解析与地理编码。
- `feed-api`：对前端提供热点查询、地图点位、信息栏推荐接口。

### 3.2 数据存储
- **PostgreSQL + PostGIS**：事件与地理查询。
- **Redis**：热点榜单缓存、短期计算结果。
- **对象存储**（可选）：原始文本快照与审计留存。

### 3.3 前端
- 地图库：Mapbox GL / Leaflet / ECharts Map。
- 状态管理：按“地图视野 + 筛选条件 + 用户偏好”请求后端推荐。
- 自动刷新：每 15 分钟拉取增量，避免整页重载。

---

## 4. 数据流（15 分钟周期）
1. 定时任务触发采集。
2. 新内容入队，执行清洗 + 去重。
3. NLP 与地理解析产出结构化事件。
4. 计算热度分与变化率。
5. 更新地图点位缓存。
6. 依据规则重排信息栏推荐。
7. 前端收到增量推送或轮询更新。

---

## 5. 关键难点与规避
- **多源噪声高**：必须做来源信誉分层 + 交叉验证。
- **地名歧义**：要有地理上下文（国家/语言/来源区域）辅助消歧。
- **实时性与成本平衡**：先 15 分钟批处理，后续再升级流式。
- **合规与版权**：优先使用授权 API 与可再分发摘要；保留来源链接。

---

## 6. MVP（4~6 周）建议范围
- 覆盖 3~5 类核心事件。
- 接入 5~10 个稳定数据源。
- 实现全球地图热点点位 + 信息栏自动排序。
- 提供基础筛选（时间、地区、事件类型）。
- 提供“突发升级”标识与来源引用。

---

## 7. 可以直接用的评分公式（示例）

```text
Hotness = 0.35 * SourceCredibility
        + 0.25 * RecencyScore
        + 0.20 * CrossSourceAgreement
        + 0.20 * SeverityScore
```

并在每个周期计算：

```text
TrendDelta = Hotness(t) - Hotness(t-1)
```

信息栏优先级可按：

```text
Priority = 0.6 * Hotness + 0.4 * TrendDelta
```

---

## 8. 你这个想法是否可行？
可行，而且很有产品价值。建议先做“可解释的规则引擎 + 人工可校验结果”的 MVP，等数据质量稳定后，再引入更复杂的模型（如事件演化预测、区域风险评分）。
